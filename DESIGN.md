# 催眠模拟 App - 设计文档

> 仿日本动漫风格的催眠App模拟器，用于剧情扮演/故事创作场景的道具。
> 纯前端实现，无需后端服务。

---

## 1. 项目概述

一个纯静态的客户端网页应用，模拟日本动漫中常见的"催眠控制App"。作为剧情扮演场景中的虚拟道具使用。应用由三个页面组成：登录页、控制面板、催眠动画。

### 技术栈

- 纯 HTML5 / CSS3 / ES6+ JavaScript
- 无框架、无构建工具、无后端
- 单页应用（SPA），基于 section 切换实现路由
- Google Fonts 加载字体 + Font Awesome 图标库
- 移动端响应式设计

### 视觉风格参考

以 `deepseek_html_20260408_d25d91.html`（天气卡片）为第一页和第二页的视觉参考，核心设计语言：

- **玫瑰红渐变背景**: `radial-gradient(circle at 20% 30%, #3f1e2b, #180b10)` + 背景光晕营造层次感
- **玻璃拟态卡片**: 半透明玫瑰底色 `rgba(40, 18, 28, 0.45)` + `backdrop-filter: blur(25px) saturate(180%)`
- **大圆角设计**: 卡片 48px 圆角，内部组件 24px 圆角
- **内发光边框**: 使用 CSS mask 技巧实现精致的边框光晕
- **卡片悬浮动画**: 6秒周期缓慢上下浮动
- **Inter 字体**: 清晰优雅的无衬线字体
- **柔和阴影层**: 多层阴影营造纵深感
- **指标网格卡片**: 半透明小卡片 + hover 交互效果

## 3. 架构设计

### 3.1 单页路由

`index.html` 包含三个 `<section>` 元素：

```
#page-login      — 登录表单
#page-dashboard  — 控制面板
#page-animation  — 催眠动画
```

同一时刻仅显示一个 section，通过 `<body>` 上的 `data-active-page` 属性控制。`app.js` 负责页面切换。

### 3.2 页面流转

```
登录页 ──[密码验证通过]──> 控制面板 ──[开始催眠按钮]──> 催眠动画
                              ^                            |
                              └───[返回 / ESC 键]──────────┘
```

### 3.3 状态管理

所有状态遵循不可变模式 —— 每次变更创建新对象，永不修改已有对象。

```
会话状态（来自登录）:
  { username, level, usageCount, verboseError, tierName, maxUsage }

设置状态（来自面板滑块）:
  { delay, intensity, obedienceDepth, sensitivity, resistance }
```

## 4. 密码系统

### 4.1 格式规范

```
ly + [2位数字等级] + [4位数字使用次数] + [1位数字错误详情开关] + [可选任意字符]
```

**正则表达式:** `^ly(\d{2})(\d{4})(\d)(.*)?$`

| 部分 | 示例 | 含义 |
|------|------|------|
| `ly` | `ly` | 固定前缀（必须） |
| `\d{2}` | `03` | 用户等级（00-99） |
| `\d{4}` | `0025` | 本月已使用次数（0000-9999） |
| `\d` | `1` | 错误详情开关：`0`=仅显示"密码错误"，`1`=显示具体错误原因 |
| `(.*)` | `abc` | 任意后缀字符（忽略） |

### 4.2 验证规则

- 用户名：任意非空字符串（去除首尾空格）
- 密码：必须精确匹配正则表达式
- 验证成功：提取等级、使用次数和错误详情开关，跳转至控制面板
- 验证失败：根据错误详情开关决定提示方式

### 4.3 错误提示

**当错误详情开关 = 1（或密码格式不完整无法提取开关值时）:**

| 条件 | 提示信息 |
|------|---------|
| 密码为空 | "请输入访问密钥" |
| 不以 `ly` 开头 | "访问密钥格式无效" |
| 等级位数不足/非数字 | "等级编码异常" |
| 使用次数位数不足/非数字 | "使用数据异常" |
| 缺少错误详情位 | "密钥配置位缺失" |
| 用户名为空 | "请输入用户名" |

**当错误详情开关 = 0:**

| 条件 | 提示信息 |
|------|---------|
| 任何密码验证失败 | "密码错误" |
| 用户名为空 | "密码错误" |

> 注：错误详情开关的判断逻辑：先尝试匹配完整正则提取开关值。若密码格式不完整（如前缀错误、位数不够），无法提取到开关值，此时默认按详情模式（=1）显示具体错误。这样只有当密码格式本身基本正确、仅细节有误时，开关=0才会生效统一显示"密码错误"。

**动画层级（从后到前）:**

1. **底色层**: 纯黑 `#000000`
2. **螺旋层**: 大尺寸 `conic-gradient` 元素，持续旋转
3. **暗角层**: 径向渐变遮罩（边缘暗、中心透明）
4. **心形层**: CSS心形，屏幕居中，带脉动和光晕
5. **粒子层**: 10-15个漂浮小圆点，随机位置和时间
6. **色调偏移**: 缓慢的 `hue-rotate` 滤镜应用于整个容器
7. **退出按钮**: 半透明 "X"，固定右上角

## 8. 动画技术规格

### 8.1 CSS 关键帧动画

| 动画名称 | 目标元素 | 动画属性 | 时序 |
|---------|---------|---------|------|
| `spiral-rotate` | 螺旋背景 | `transform: rotate()` | linear infinite |
| `spiral-pulse` | 螺旋背景 | `transform: scale()` | ease-in-out infinite |
| `heartbeat` | 心形 | `transform: scale()` | 自定义贝塞尔 infinite |
| `heart-glow` | 心形 | `box-shadow` | 与心跳同步 |
| `fade-in` | 动画容器 | `opacity` | 黑屏后执行一次 |
| `float-particle` | 粒子 | `transform: translate()` | linear infinite 各不同 |
| `hue-shift` | 遮罩层 | `filter: hue-rotate()` | linear infinite 缓慢 |
| `blackout-text` | 状态文字 | `opacity` | 淡入淡出序列 |
| `card-float` | 登录/面板卡片 | `transform: translateY()` | ease-in-out 6s infinite |

### 8.2 强度分级

| 强度 | 旋转速度 | 心跳速度 | 颜色饱和度 |
|------|---------|---------|----------|
| 1-3（低） | 8秒/圈 | 1.5秒/拍 | 60% 不透明度，柔和 |
| 4-6（中） | 5秒/圈 | 1.2秒/拍 | 80% 不透明度，正常 |
| 7-9（高） | 3秒/圈 | 0.8秒/拍 | 100% 不透明度，鲜艳 |
| 10（极限） | 2秒/圈 | 0.6秒/拍 | 120% 饱和度，最大光晕 |

### 8.3 心形实现（CSS）

使用经典的"两个旋转圆角矩形"技巧：

```css
.heart {
  position: relative;
  width: 100px;
  height: 90px;
}
.heart::before,
.heart::after {
  content: '';
  position: absolute;
  width: 50px;
  height: 80px;
  background: var(--accent-pink);
  border-radius: 50px 50px 0 0;
}
.heart::before { left: 50px; transform: rotate(-45deg); transform-origin: 0 100%; }
.heart::after  { left: 0;    transform: rotate(45deg);  transform-origin: 100% 100%; }
```

### 8.4 螺旋背景实现（CSS）

```css
.spiral {
  width: 200vmax;
  height: 200vmax;
  background: conic-gradient(
    from 0deg,
    var(--accent-pink) 0deg,
    transparent 30deg,
    var(--accent-purple) 60deg,
    transparent 90deg,
    /* ... 重复段落 */
  );
  animation: spiral-rotate var(--rotation-speed) linear infinite;
}
```

### 8.5 性能规则

- 仅动画 `transform` 和 `opacity`（GPU合成属性）
- 在动画元素上添加 `will-change: transform`
- 粒子数量上限 15 个（移动端限制 10 个）
- 动画中不使用触发回流的属性
- 使用 CSS 自定义属性让 JS 控制动画速度

## 10. 风险与缓解

| 风险 | 严重程度 | 缓解方案 |
|------|---------|---------|
| `conic-gradient` 浏览器兼容性 | 低 | 所有现代浏览器均支持；添加径向渐变降级方案 |
| 心形CSS跨浏览器不一致 | 低 | 使用成熟的旋转方块技巧；必要时使用SVG备选方案 |
| 滑块样式跨浏览器不一致 | 中 | 显式编写WebKit + Firefox伪元素样式 |
| 低端移动设备动画卡顿 | 中 | 仅动画transform/opacity；移动端减少粒子数量 |
| Google Fonts 加载缓慢 | 低 | `font-display: swap` + 系统字体备选 |
| `backdrop-filter` 兼容性 | 低 | 添加 `-webkit-backdrop-filter` 前缀；不支持时降级为纯色背景 |
